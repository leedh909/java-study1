*24-5 Random Access 파일과 FILE 클래스
스트림?->(=흐름)-> ex) A에서 B로 흐를때 순차적이다.
//파일과 입출력할때 순차적으로 읽어오고 출력한다.
Random Access는 I/O랑 별개의 파트로 봐도 괜찮다.
RandomAccessFile 클래스는 입출력 위치를 임의로 변경할 수 있다.!!!!!!(파일을 대상으로만 존재)
getFilPointer()메소드로 어디까지 읽었는지를 반환해주는 메소드이다.
seek()메소드는 현재어디를 읽을 것인지, 어디까지 썼느냐를 반환해주는 메소드이다.
생성모드로 읽기위한용도인지,읽고쓰기위한용도인지 판단.
File 클래스 -> 디렉터리의 (생성, 소멸) , 파일의 (생성,소멸), 디렉터리 내에 존재하는 파일이름 출력.
//File myFile = new File(파일정보);
renameTo()메소드는 파일의 이름을 변경하는 메소드인데 경로까지 변경(이동)이 가능하다.
File.separator는 운영체제에 따른 구분자로 각각 치환된다.(ex) 윈도우는 || 인데 리눅스는 / 이다.
